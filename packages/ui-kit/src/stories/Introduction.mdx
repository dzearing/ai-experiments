# Claude Flow UI Kit

Welcome to the Claude Flow UI Kit - a framework-agnostic foundation for building consistent, themeable user interfaces.

## Overview

The UI Kit provides:

- üé® **15 Built-in Themes** - Each with light and dark modes (30 total variations)
- üîß **CSS Variables** - Comprehensive design tokens for colors, typography, spacing, and more
- üìê **CSS Reset** - Consistent cross-browser baseline
- üöÄ **Zero JS Runtime** - Pure CSS with inline critical styles for instant theming
- ‚ôø **Accessible** - WCAG AA/AAA compliant color contrasts and motion preferences
- üì± **Responsive** - Mobile-first design tokens
- üéØ **Framework Agnostic** - Works with React, Vue, Angular, or vanilla JS

## Quick Start

### 1. Installation

```bash
pnpm add @claude-flow/ui-kit
```

### 2. Import in Your Application

#### Option A: HTML (Vanilla JS)

```html
<head>
  <!-- CSS Reset, Variables, and Base Styles -->
  <link rel="stylesheet" href="node_modules/@claude-flow/ui-kit/dist/styles.css" />

  <!-- Theme initialization (prevents flash of wrong theme) -->
  <script src="node_modules/@claude-flow/ui-kit/dist/theme-init.js"></script>
</head>
```

#### Option B: JavaScript Module Import

```javascript
// Import CSS in your entry point
import '@claude-flow/ui-kit/styles.css';

// Import theme manager for programmatic control
import { themeManager } from '@claude-flow/ui-kit';
```

#### Option C: React/TypeScript

```typescript
// In your App.tsx or index.tsx
import '@claude-flow/ui-kit/styles.css';
import { themeManager, type Theme } from '@claude-flow/ui-kit';

// Initialize on mount
useEffect(() => {
  // Theme is auto-initialized, but you can manually control it
  themeManager.setTheme('ocean');
}, []);
```

### 3. Use CSS Variables

```css

  border: 1px solid var(--color-border);
  box-shadow: var(--shadow-sm);

```

## Theme Switching

### JavaScript API

```javascript
// Using the global theme API (available after theme-init.js loads)
window.__claudeFlowTheme.loadTheme('ocean', 'dark');

// Or use the imported theme manager
import { themeManager } from '@claude-flow/ui-kit';

// Switch theme
themeManager.setTheme('sunset');

// Switch mode (light/dark/auto)
themeManager.setMode('dark');

// Toggle between light and dark
window.__claudeFlowTheme.toggleThemeType();

// Listen for theme changes
themeManager.on('themechange', (event) => {
  console.log(`Theme changed to ${event.theme} (${event.mode})`);
});
```

### React Example

```tsx
import React, { useState, useEffect } from 'react';
import { themeManager, type ThemeChangeEvent } from '@claude-flow/ui-kit';

export function ThemeSwitcher() {
  const [currentTheme, setCurrentTheme] = useState(themeManager.currentTheme);
  const [currentMode, setCurrentMode] = useState(themeManager.currentMode);

  useEffect(() => {
    // Listen for theme changes
    const handleThemeChange = (event: ThemeChangeEvent) => {
      setCurrentTheme(event.theme);
      setCurrentMode(event.mode);
    };

    themeManager.on('themechange', handleThemeChange);
    return () => themeManager.off('themechange', handleThemeChange);
  }, []);

  return (
    <div>
      <select 
        value={currentTheme} 
        onChange={(e) => themeManager.setTheme(e.target.value)}
      >
        {themeManager.themes.map(theme => (
          <option key={theme.id} value={theme.id}>
            {theme.name}
          </option>
        ))}
      </select>

      <button onClick={() => themeManager.setMode('light')}>Light</button>
      <button onClick={() => themeManager.setMode('dark')}>Dark</button>
      <button onClick={() => themeManager.setMode('auto')}>Auto</button>
    </div>
  );
}
```

### TypeScript Types

```typescript
import type { Theme, ThemeMode, ThemePreferences } from '@claude-flow/ui-kit';

// Theme interface
interface Theme {
  id: string;
  name: string;
  description: string;
  colors: {
    primary: string;
    secondary: string;
  };
}

// Theme mode options
type ThemeMode = 'light' | 'dark' | 'auto';

// Stored preferences
interface ThemePreferences {
  theme: string;
  mode: ThemeMode;
  timestamp: number;
}
```

## Design Principles

### 1. **Semantic First**

Every design token has meaning. Colors aren't just "blue-500" but have semantic purposes like `--color-primary` or `--color-success`.

### 2. **Scalable Systems**

Our scales (colors, spacing, typography) follow mathematical progressions for visual harmony.

### 3. **Performance Focused**

CSS variables enable instant theme switching without JavaScript frameworks or re-renders.

### 4. **Accessibility Built-in**

- All color combinations meet WCAG AA standards (High Contrast theme meets AAA)
- Respects `prefers-reduced-motion`
- Semantic HTML encouraged through reset styles

### 5. **Developer Experience**

- Comprehensive documentation
- Interactive examples
- Full TypeScript support
- Visual theme explorer

## Advanced Usage

### Preloading Themes

```javascript
// Preload themes for instant switching
window.__claudeFlowTheme.preloadThemes(['ocean', 'sunset', 'forest']);

// Or preload specific mode combinations
window.__claudeFlowTheme.preloadThemes([
  { name: 'ocean', type: 'dark' },
  { name: 'sunset', type: 'light' }
]);
```

### Custom CSS with Tokens

```css
/* Use color scales directly */
.custom-gradient {
  background: linear-gradient(
    to right,
    var(--color-primary-400),
    var(--color-primary-600)
  );
}

/* Responsive spacing */
.responsive-padding {
  padding: var(--spacing-sm);
}

@media (min-width: 768px) {
  .responsive-padding {
    padding: var(--spacing-md);
  }
}

/* Dark mode overrides */
[data-theme-type="dark"] .special-element {
  background: var(--color-surface-elevated);
  border-color: var(--color-border-subtle);
}
```
